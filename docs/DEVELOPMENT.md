# Development guide

Everything needed to pick up Wordplay in a fresh session. Pair this with
`README.md` (what the app is) and `docs/BUILD-LOG.md` (how it came to be).

## Orient in 30 seconds

```sh
swift test        # 38 tests, all green
swift build       # debug build
./package.sh      # → dist/Wordplay.app (ad-hoc signed, double-clickable)
swift run         # launch from source
```

A clean clone builds and tests with no extra setup — the CMU Pronouncing
Dictionary and ENABLE word list are committed under
`Sources/Anagrammer/Resources/`.

## Layout

- **`Sources/AnagramEngine/`** — the pure, UI-free, tested engine. Every
  feature's logic lives here so it can be unit-tested and reused (including by
  the web ports). Key files: `AnagramEngine`, `RackSolver`, `PatternMatcher`,
  `WordLadder`, `Cryptic`, `Phonetics` (CMUdict parser), `Fusion` (phonetic
  overlap + audible-bonus scan), `Connections` (the seven-dimension word-web),
  `PathFinder` (six-degrees bidirectional BFS), `MinimalPairs` (one-phoneme
  neighbours + ARPABET distinctive-feature labelling).
- **`Sources/Anagrammer/`** — the SwiftUI front end. One `*View.swift` per
  sidebar tool; `AnagrammerApp.swift` owns the `Tool` enum + sidebar routing;
  `WordStore` loads the dictionary once and vends every engine (CMUdict and the
  heavier finders load lazily, off-main, cached); `ChimeEngine` does the
  synthesised chimes and speech; `Components.swift` has shared views
  (`Field`, `ToolScaffold`).
- **`web/`** — single-file HTML ports (`fusions.html`, `codex.html`): the
  engine re-implemented in JS with the lexicon embedded, so they run offline in
  any browser on any OS. Regenerated by a bake step that inlines the dictionary
  (see the build log); the Swift engine is the source of truth — keep the JS in
  step when you change engine behaviour.

## Adding a tool (the established pattern)

1. Put the logic in `Sources/AnagramEngine/` with tests in
   `Tests/AnagramEngineTests/`.
2. If it needs a prebuilt index over the dictionary, add a cached
   `xFinder() async -> …?` to `WordStore` (mirror `fusionFinder()` /
   `minimalPairFinder()` — build off-main, cache, reuse).
3. Add a `SomethingView.swift` under `Sources/Anagrammer/`.
4. Register it in `AnagrammerApp.swift`: a `Tool` case, an SF Symbol in
   `icon`, an entry in `toolsSection`, and a line in the detail `switch`.

## Conventions

- **Version every release.** Bump `VERSION` in `package.sh` (single source of
  truth → Info.plist → shown in the sidebar footer).
- **Keep the engine pure.** No SwiftUI or AppKit imports in `AnagramEngine`.
- **Heavy work off-main.** Dictionary-scale searches run on detached tasks so
  the UI stays responsive; cache prebuilt indexes on `WordStore`.
- Tests use the `swift-testing` framework (`@Test` / `#expect`), not XCTest.

## Design direction for the "Web" tool (the codex)

This is the app's signature view and its look was hard-won — six iterations,
documented in the build log. The settled register, **do not drift from it
without asking**:

- **The codex** (default): sigil circles chained across aged parchment;
  connections written in curved script around hand-drawn (deliberately
  wobbling) rings; the seven relation dimensions marked with the seven
  classical planetary glyphs (☉ ☽ ☿ ♀ ♂ ♃ ♄); mirror-written margin notes.
  Mood: **esoteric, mystical, da-Vinci-notebook** — *not* churchy (no scripture
  voice, no verses). It's a living document: it writes itself, its ink ages and
  pales, the oldest unopened words fade and are reaped, it persists across
  launches, and it can speak words aloud.
- **The crossword** (toggle): the same web as a self-writing crossword on an
  endless grid, words physically crossing at shared letters, crossings tinted
  by dimension.
- Hard-won lesson: when a visual feels wrong, **change the object, not the
  skin.** A string of dark-void "glowing dots and lines" reskins all read the
  same; the crossword and codex worked because they changed *what the words
  are*, not the palette.

The web `codex.html` mirrors all of this and is touch-ready (tap to open, drag
to pan, pinch to zoom, controls fold away on phones).

## Roadmap / open threads

- **Phonology mode** (in progress): `MinimalPairs` (done) is part 1. Natural
  next pieces — *phonotactic legality scoring* so Fusions only yields
  pronounceable pseudo-words, and *feature-weighted fuzzy fusions* where near
  phonemes (m~n, ɪ~i) count as weak overlaps ranked below exact ones.
- The connection web still mixes an orthographic relation (one-letter
  Word-Ladder step) with phonemic ones; a phonology-purist mode could swap in
  minimal pairs.
- From the original README wishlist: curated Scrabble word list option, more
  languages, a haiku/limerick helper on the syllable counter, a spoonerism
  generator on the phonetic data.
