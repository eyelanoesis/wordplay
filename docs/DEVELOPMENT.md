# Development guide

Everything needed to pick up Wordplay in a fresh session. Pair this with
`README.md` (what the app is) and `docs/BUILD-LOG.md` (how it came to be).

## Where things live (since 2026-07-24)

**`app/` is the live codebase.** TypeScript is the single source of truth for
all wordplay logic; the web is the primary platform. Desktop (Tauri) and
mobile (Capacitor) wrappers come later — nothing blocks them.

- **`Sources/` + `Package.swift` + `package.sh`** — the original native macOS
  SwiftUI app, **frozen at v7.2.0**. It still builds (`swift test`, 40 green;
  `./package.sh`) and stays as the reference implementation and the way back
  to native if ever wanted. Don't develop here.
- **`web/codex.html`, `web/fusions.html`** — frozen legacy single-file
  artifacts, never edited again. New offline builds come from
  `npm run build:single`.
- **`tools/assoc-dump/`** — standalone Swift CLI (macOS-only, run at build
  time) that dumps NLEmbedding semantic neighbors to data files so the
  Association (♅) dimension works on every platform.

## Orient in 30 seconds

```sh
cd app
npm install
npm test            # tsc --noEmit + vitest: 43 tests (the 40 Swift-suite ports + 3 full-data)
npm run dev         # Vite dev server → the codex at localhost:5173
npm run build       # hosted build → app/dist/
npm run build:single  # ONE offline HTML file → app/dist-single/index.html (~3.6MB)
```

## Layout

- **`app/src/engine/`** — the pure, tested engine; one file per frozen Swift
  twin in `Sources/AnagramEngine/` (letterCount, wordList, anagramEngine,
  scrabble, rackSolver, patternMatcher, wordLadder, phonetics, fusion,
  cryptic, minimalPairs, connections, pathFinder) plus `associations.ts`
  (parses the assoc dumps). No I/O, no DOM — callers hand in text.
- **`app/src/codex/`** — the codex UI, modularized from the legacy
  codex.html with behavior preserved: `dataSource` (fetch in dev, `?raw` in
  the single-file build), `relMeta` (glyphs/colors/explanations), `settings`
  + `model` (localStorage keys `wordplay-codex-settings` / `wordplay-codex`,
  shapes unchanged), `engineAdapter` (lazy per-dictionary ConnectionWeb +
  PathFinder), `sound`, `render`, `input`, `hud`, `styles.css`.
- **`app/tests/`** — 7 vitest files mirroring `Tests/AnagramEngineTests/`
  (all 40 test names preserved; fixtures copied verbatim) + `realData.test.ts`
  (full ENABLE + lexicon; pins the canonical `fire → firm → term → water`).
- **`app/public/data/`** — committed data: `enable/biblical/dance.txt` +
  `cmudict.dict` (synced from the frozen app's Resources by
  `npm run sync-data`), `lexicon.txt` (generated by `npm run build-lexicon`),
  `assoc-*.txt` (generated by tools/assoc-dump).

## Conventions

- **The engine stays pure and 1:1.** Each engine file mirrors its Swift twin
  so parity review is a side-by-side diff. Port fixes that change behavior go
  in the commit message (the "divergence ledger") — e.g. the legacy JS
  schwaVariants double-push and the missing fusion sort tiebreak were fixed
  in the port.
- **Tests are the spec.** New engine behavior lands with tests; the Swift
  suite is not edited (the app is frozen).
- **TypeScript strict; zero runtime deps.** Four dev deps (vite, typescript,
  vitest, vite-plugin-singlefile).
- **Every session ends committed and pushed** to `origin/master` — the GitHub
  repo is the source of truth.
- Engine budgets (dev Mac, full ENABLE): codex expand < 100ms steady-state
  (currently ~13ms; the first expand per dictionary pays a ~300ms lazy engine
  build), boot to interactive < 1.5s.

## Data pipeline

`Sources/Anagrammer/Resources/` stays the canonical home of the dictionaries;
`npm run sync-data` copies them into `app/public/data/`. `npm run
build-lexicon` regenerates `lexicon.txt` (`word|PHONES|RHYMEKEY`, ENABLE ∩
cmudict, ~51k entries). Association dumps regenerate with:

```sh
swift run --package-path tools/assoc-dump -c release assoc-dump \
  app/public/data/enable.txt app/public/data/assoc-enable.txt \
  --limit-to app/public/data/lexicon.txt --n 12
```

(A Mac is required — NLEmbedding is Apple-only; that's the point of dumping.)
The single-file build currently ships **without** the 4.9MB enable dump; the
dev/hosted app loads it by fetch. Embedding it (plain or gzip+base64 via
DecompressionStream) is an open decision.

## Design direction for the codex

The codex's look was hard-won — six iterations, documented in the build log.
The settled register, **do not drift from it without asking**: sigil circles
chained across aged parchment; connections in curved script around
hand-wobbled rings; the nine dimensions marked with planetary glyphs
(☉ ☽ ☿ ♀ ♂ ♃ ♄ ♅ ♆); mirror-written margin notes; frontier ink (verdigris =
unopened, sepia = opened, red ochre = the codex's own hand). Mood: esoteric,
mystical, da-Vinci-notebook — *not* churchy. It writes itself, its ink ages,
the oldest unopened words fade and are reaped, it remembers across launches,
and it can speak. Hard-won lesson: when a visual feels wrong, **change the
object, not the skin.**
